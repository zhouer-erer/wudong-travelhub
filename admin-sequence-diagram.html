sequenceDiagram
    participant 前端 as 前端页面
    participant Controller as AuthController
    participant Service as AdminService
    participant DB as MySQL
    participant Cache as smsStore(内存)

    前端->>Controller: POST /api/auth/login
{username, password}
    Controller->>Service: findByUsername(username)
    Service->>DB: SELECT * FROM admin WHERE username=?
    DB-->>Service: 返回admin记录
    Service-->>Controller: 返回admin实体

    alt 用户名不存在
        Controller-->>前端: 401 用户名或密码错误
    end

    alt 账号已禁用
        Controller-->>前端: 403 账号已被禁用
    end

    alt 账号被锁定
        Controller-->>前端: 423 账号已被锁定，请X分钟后再试
    end

    Controller->>Controller: bcrypt.compare(password, hash)

    alt 密码错误
        Controller->>Service: update(login_fail_count++)
        Service->>DB: UPDATE admin SET login_fail_count=?
        Controller-->>前端: 401 用户名或密码错误，剩余X次机会
        
        alt 连续5次错误
            Controller->>Service: update(locked_until=30分钟后)
            Service->>DB: UPDATE admin SET locked_until=?
            Controller-->>前端: 423 密码错误次数过多，账号已锁定30分钟
        end
    end

    alt 密码正确
        Controller->>Service: update(login_fail_count=0)
        Service->>DB: UPDATE admin SET login_fail_count=0
        Controller->>Controller: 生成6位验证码
生成smsToken(UUID)
        Controller->>Cache: set(smsToken, {code, adminId, expiresAt})
        Controller-->>前端: 200 {needSms, phoneMask, smsToken}
    end

    前端->>Controller: POST /api/auth/verify-sms
{smsToken, code}
    Controller->>Cache: get(smsToken)

    alt 验证码过期或无效
        Cache-->>Controller: null/过期
        Controller->>Cache: delete(smsToken)
        Controller-->>前端: 401 验证码已过期，请重新登录
    end

    Cache-->>Controller: 返回存储数据
    Controller->>Cache: delete(smsToken)
    Controller->>Controller: JWT.sign({id, username, role_id})
    Controller->>Service: findById(adminId)
    Service->>DB: SELECT * FROM admin WHERE id=?
    DB-->>Service: 返回admin记录
    Service-->>Controller: 返回admin实体
    
    Controller->>Controller: 查询登录IP地点
检测异地登录
    alt 异地登录
        Controller->>Controller: 发送系统消息提醒
    end
    
    Controller->>Service: update(last_login_at, last_login_ip, ...)
    Service->>DB: UPDATE admin SET last_login_*=?
    Controller-->>前端: 200 {token, admin}
        
sequenceDiagram
    participant 前端 as 前端页面
    participant Middleware as 认证中间件
    participant JWT as JwtService
    participant Service as AdminService
    participant DB as MySQL

    前端->>Middleware: HTTP请求 + Authorization: Bearer {token}
    Middleware->>JWT: verify(token)
    
    alt Token无效或过期
        JWT-->>Middleware: 抛出异常
        Middleware-->>前端: 401 token无效或已过期
    end

    JWT-->>Middleware: 返回payload {id, username, role_id, role}
    
    alt 商家Token (role=merchant)
        Middleware->>Middleware: 查询商家表验证
        Middleware->>Middleware: 检查强制下线(Redis)
        alt 商家账号禁用/强制下线
            Middleware-->>前端: 401 商家账号已被禁用/强制下线
        end
        Middleware->>Middleware: ctx.state.merchant = {...}
    else 管理员Token
        Middleware->>Service: findById(id)
        Service->>DB: SELECT * FROM admin WHERE id=?
        DB-->>Service: 返回admin记录
        Service-->>Middleware: 返回admin实体
        
        alt 管理员账号禁用
            Middleware-->>前端: 401 账号已被禁用
        end
        
        Middleware->>Middleware: ctx.state.admin = {...}
    end
    
    Middleware->>Middleware: 继续执行下一个中间件
    Note right of Middleware: RBAC权限校验 -> Controller -> Response
        
sequenceDiagram
    participant Middleware as RBAC中间件
    participant Redis as Redis缓存
    participant PermissionService as PermissionService
    participant DB as MySQL

    Middleware->>Middleware: 检查白名单路由
    
    alt 在白名单中
        Middleware->>Middleware: 直接放行
    end

    Middleware->>Middleware: 检查商家请求
    
    alt 商家访问非商家接口
        Middleware-->>前端: 403 商家账号无权访问此接口
    end

    Middleware->>Middleware: 最长前缀匹配路由权限码
    Middleware->>Middleware: ROUTE_PERMISSION_MAP查找
    
    alt 路由不在映射表中
        Middleware->>Middleware: 放行(fail-open)
    end

    Middleware->>Redis: get(rbac:permissions:{roleId})
    
    alt Redis命中
        Redis-->>Middleware: 返回权限码数组
    else Redis未命中/异常
        Middleware->>PermissionService: findByRoleId(roleId)
        PermissionService->>DB: SELECT * FROM permission
JOIN role_permission
WHERE role_id=?
        DB-->>PermissionService: 返回权限列表
        PermissionService-->>Middleware: 返回权限码数组
        Middleware->>Redis: set(rbac:permissions:{roleId}, EX=300)
    end

    Middleware->>Middleware: 检查权限码是否包含
    
    alt 权限不足
        Middleware-->>前端: 403 权限不足，无法执行此操作
    end

    Middleware->>Middleware: 放行请求
    Note right of Middleware: Controller处理业务逻辑 -> 返回响应
